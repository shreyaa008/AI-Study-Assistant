# API Documentation — AI Study Assistant

Base URL: `https://<your-api-id>.execute-api.us-east-1.amazonaws.com/dev`

All endpoints are `POST`, accept and return `application/json`, and are backed by Lambda proxy integration with CORS enabled.

---

## POST /upload

Uploads a study note and stores it.

**Request body**
```json
{
  "student_id": "shreya01",
  "filename": "os-notes.txt",
  "file_content_base64": "<base64-encoded file content>"
}
```

**Success response — 200**
```json
{
  "message": "Upload successful",
  "note_id": "19b04643-01f4-4bc9-9c38-9eb440bab577"
}
```

**Error responses**
- `400` — missing `student_id`, `filename`, or `file_content_base64`
- `500` — internal error (see CloudWatch logs for `upload_handler`)

---

## POST /summarize

Generates a bullet-point summary of a previously uploaded note.

**Request body**
```json
{
  "student_id": "shreya01",
  "note_id": "19b04643-01f4-4bc9-9c38-9eb440bab577"
}
```

**Success response — 200**
```json
{
  "note_id": "19b04643-01f4-4bc9-9c38-9eb440bab577",
  "summary": "- Point one\n- Point two\n- Point three"
}
```

**Error responses**
- `400` — missing fields, or the note isn't a `.txt` file
- `404` — no note found for that `student_id` + `note_id`
- `500` — internal error (see CloudWatch logs for `summarize_handler`)

## POST /ask

Answers a question, grounded only in the content of one specific note.

**Request body**
```json
{
  "student_id": "shreya01",
  "note_id": "19b04643-01f4-4bc9-9c38-9eb440bab577",
  "question": "What is the difference between a semaphore and a mutex?"
}
```

**Success response — 200**
```json
{
  "note_id": "19b04643-01f4-4bc9-9c38-9eb440bab577",
  "question": "What is the difference between a semaphore and a mutex?",
  "answer": "Based on the notes, a mutex..."
}
```

Every Q&A pair is also saved to DynamoDB as history (`SK = QA#<timestamp>`).

**Error responses**
- `400` — missing fields, or the note isn't a `.txt` file
- `404` — note not found
- `500` — internal error (see CloudWatch logs for `ask_handler`)

## POST /quiz

Generates 5 multiple-choice quiz questions from a note.

**Request body**
```json
{
  "student_id": "shreya01",
  "note_id": "19b04643-01f4-4bc9-9c38-9eb440bab577"
}
```

**Success response — 200**
```json
{
  "note_id": "19b04643-01f4-4bc9-9c38-9eb440bab577",
  "quiz": [
    {
      "question": "What does IPC stand for?",
      "options": ["Inter-Process Communication", "Internal Program Control", "Instruction Pipeline Cache", "Input Processing Chain"],
      "correct_answer": "A"
    }
  ]
}
```

**Error responses**
- `400` — missing fields, or unsupported file type
- `404` — note not found
- `500` — internal error, including `"The AI response couldn't be understood"` if Claude's output didn't parse as valid JSON (see CloudWatch logs for `quiz_flashcard_handler`)

## POST /flashcards

Generates 8 flashcards (front/back) from a note. Served by the same Lambda as `/quiz` — routed internally based on the request path.

**Request body**
```json
{
  "student_id": "shreya01",
  "note_id": "19b04643-01f4-4bc9-9c38-9eb440bab577"
}
```

**Success response — 200**
```json
{
  "note_id": "19b04643-01f4-4bc9-9c38-9eb440bab577",
  "flashcards": [
    { "front": "Semaphore", "back": "A synchronization primitive that controls access to a shared resource using a counter." }
  ]
}
```

**Error responses** — same as `/quiz`.

---

## Common error shape

Every endpoint returns errors in the same shape:
```json
{ "error": "Human-readable message" }
```

## Notes on data flow

- `note_id` is generated by `/upload` and must be reused for every subsequent call about that note
- All AI-generating endpoints (`/summarize`, `/ask`, `/quiz`, `/flashcards`) currently support `.txt` files only
- Generated content (summaries, quizzes, flashcards) is persisted to DynamoDB, not just returned once — future requests could check for an existing result before regenerating (not yet implemented in v1)
